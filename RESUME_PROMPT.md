# 🌅 新会话开场咒语

> 当你结束一个 AI 会话、第二天(或几天后)开启**新会话**时,复制下面这段话作为第一条消息发给 AI。
> 它会让新 AI 读完项目档案,无缝接上之前的进度。

---

## 📋 复制即用的开场咒语

```
请打开项目目录:/Users/alhao/Documents/Vibe Coding/Best Practices/mathplay

然后先读这 3 个文件,按顺序:
1. SPEC.md(项目宪法,知道我们在做什么)
2. PROGRESS.md(进度档案,知道做到哪一步)
3. DECISIONS.md(决策记录,知道为什么这么选)

读完之后:
- 用一段话总结当前状态(在哪一步、下一步是什么)
- 然后停下等我指示,不要主动开工

补充信息:我是 vibe coding 新手,正在为 8 岁女儿做一个学习"时针分针联动"的 iPad 网页。
```

---

## 🎯 发完咒语后,根据情况选择下一句

### 情况 A:已让女儿试玩过(推荐进入 Step 5 Phase B)
```
女儿已经试玩过了,以下是我的观察:
- [她最先做了什么动作]
- [卡在哪里 / 哪里有"咦?"的反应]
- [自由玩 vs 挑战哪个更吸引她]
- [通关时的反应]
- [是否问过"为什么时针也会动"]

开始 Step 5 Phase B 打磨。
```

### 情况 B:还没让女儿玩
```
还没让女儿玩。先帮我准备一份试玩观察清单,等观察完再继续。
```

### 情况 C:想跳过观察直接做 Step 5 通用打磨
```
先不试玩了,直接根据 Step 5 候选清单挑几个最有价值的打磨项,
讲思路给我听,我拍板后再开工。
```

---

## 💡 为什么需要这个咒语?

AI 每次会话都是"失忆"状态——没有上次的记忆。但我们已经把所有上下文都写进了 3 个 .md 文件:
- **SPEC.md** = 项目宪法(做什么、为谁做、范围、原则)
- **PROGRESS.md** = 当前进度(在哪一步、下一步是什么、技术小抄)
- **DECISIONS.md** = 决策记录(为什么选 A 不选 B,避免循环讨论)

新 AI 读完这 3 个文件,就能 100% 接力,不需要你重新解释一遍。

---

## 🔑 关键提示词原则(给未来的你看)

1. **"先读,后停"** — 让 AI 先读文档,**主动停下等指示**,而不是读完就猜你的意图开工
2. **明确身份** — 提醒它"我是 vibe coding 新手"(它会调整解释风格)
3. **明确目标对象** — "为 8 岁女儿做"(它会记得儿童友好这个最高约束)
4. **不要急** — 让 AI 总结当前状态给你看,确认它真的理解了再开工
